# 프로젝트 작업 계획

## 기본 작업 원칙

- 작업을 시작하기 전에 변경 계획을 먼저 정리한다.
- 앞으로 프로젝트에서 해야 할 내용과 실제 적용한 내용은 이 `skill.md`에 계속 기록한다.
- CSS 파일은 `css` 폴더에 배치한다.
- 페이지별 화면 스타일은 페이지마다 CSS 파일을 하나씩 분리해서 관리한다.
- CSS 클래스명은 `_`를 사용하는 스네이크 표기법으로 작성한다.
- JavaScript 또는 TypeScript 식별자와 함수명은 의미가 직관적으로 드러나는 카멜 표기법으로 작성한다.

## 스타일 시스템 계획

- 전역 색상과 폰트 시스템은 `:root`를 활용해 관리한다.
- 이미 존재하는 `:root` 변수는 우선적으로 재사용한다.
- 새 색상이 필요한 경우 기존 `:root` 변수에 없는 색상만 `hex` 값으로 직접 작성한다.
- 공통 스타일은 중복 작성하지 않고 재사용 가능한 구조로 정리한다.

## 1차 적용 계획

1. 현재 `src` 내부 파일 구조를 확인한다.
2. 기존 CSS 파일과 스타일 import 경로를 파악한다.
3. CSS 파일을 `css` 폴더 기준으로 정리할 수 있는지 확인한다.
4. `:root`에 정의된 컬러와 폰트 변수를 확인한다.
5. 기존 클래스명을 스네이크 표기법 기준으로 정리할 범위를 확인한다.
6. 실제 변경 전에 변경 대상 파일과 영향 범위를 다시 기록한다.
